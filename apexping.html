Context-Sensitive Ping System (Apex Legends Study)

Gameplay Systems · UI Architecture
(Personal Project)

Project Overview

A systems-focused recreation of Apex Legends' context-sensitive ping system, rebuilt in Unity to explore scalable UI architecture, world-space and screen-space synchronization and input-driven state transitions.

The goal of this project was not to visually replicate the feature but to architect a modular ping framework capable of handling contextual input, off-screen projection logic and radial selection mechanics.

Role & Production Details

Role: Gameplay Systems & UI Programmer
Engine: Unity (C#)
Platform: PC
Duration: 4 Months (Personal Technical Study)
Focus: Systems Architecture · Input Handling · UI/World Synchronization

Core Implementation Areas

  • Architected a modular ping management system supporting contextual ping types
  • Designed dual-camera UI setup separating world-space and screen-space canvases
  • Implemented off-screen projection using normalized screen-space vectors
  • Built radial ping wheel system driven by hold/release input state transitions
  • Implemented billboard logic to maintain world-space UI readability
  • Developed ping lifetime and priority handling logic
  • Integrated Unity's New Input System with interaction-based bindings

Core Systems

World & Screen Space Ping Architecture

Implemented dual-camera setup with separate canvases for world-space and overlay UI. Off-screen pings are projected using normalized direction vectors and clamped to a fixed-radius boundary in screen space.

Context-Sensitive Input Handling

Built using Unity’s New Input System with interaction based triggers (Press vs Release). Hold input activates radial selection mode, while release input spawns contextual pings based on state evaluation.

Ping Lifecycle & Priority System

Designed ping objects with time-based destruction and priority-aware display logic to ensure higher-importance pings override less critical ones when off-screen.

Technical Challenges & Solutions

  • Synchronizing World-Space and Screen-Space UI: Solved by separating rendering responsibilities across dedicated cameras and using coordinate transformation to project world positions into UI space.
  • Off-Screen Ping Accuracy: Calculated direction vectors from screen center to projected world position, normalized and clamped to a defined radius to maintain consistent UI placement.
  • Input State Conflicts (Tap vs Hold): Resolved using interaction-based bindings in Unity’s New Input System to differentiate between quick ping actions and ping wheel activation.
  • Maintaining UI Readability in 3D Space: Implemented billboard rotation logic aligning world-space UI with camera orientation.
