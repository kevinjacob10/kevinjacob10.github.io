Project Overview

Circuit Breakout: Dual Polarity is a fast-paced action game centered around a dynamic polarity-switching combat mechanic. Players must strategically shift between two energy states to counter enemy types, navigate hazards, and optimize combat flow.

The project focused heavily on systemic design, responsiveness, and scalable gameplay architecture to support future content expansion and commercial release.

My Responsibilities

  • Designed and implemented the core dual polarity combat system
  • Developed a modular ability framework supporting scalable mechanics
  • Built enemy AI architecture using state-based behavior systems
  • Implemented combat feedback systems (VFX triggers, UI cues, hit responses)
  • Optimized gameplay systems for performance stability and responsiveness

Core Systems

Dual Polarity System

The core mechanic allows the player to switch between two energy states to counter specific enemy types. Matching polarity increases damage output, while mismatched polarity reduces effectiveness.

The system was architected using a modular state manager, allowing clean separation between player logic, combat calculations, and feedback systems. This ensured rapid iteration and future extensibility.

Ability System Architecture

Designed a reusable ability framework that supports cooldown-based actions, conditional triggers, layered effects, and scalable expansion.

Abilities were built using a data-driven approach to reduce hardcoding and improve designer iteration speed.

Technical Challenges

Responsive State Switching

Ensuring polarity switching felt instantaneous without breaking animation flow required decoupling gameplay logic from animation states.

Implemented buffered input handling and state validation checks to maintain responsiveness under high-action scenarios.

System Scalability

Early prototypes exposed limitations in tightly coupled systems. Refactored core logic into modular components to allow expansion without architectural rewrites.

Combat Feedback Clarity

Balancing mechanical depth with player readability required clear audiovisual feedback for polarity interactions and damage states.

Outcome & Learnings

Circuit Breakout strengthened my approach to modular system design, scalable architecture, and gameplay clarity. The project reinforced the importance of decoupled systems for rapid iteration and long-term maintainability.

It also highlighted the balance required between mechanical complexity and player readability in action-driven gameplay systems.
